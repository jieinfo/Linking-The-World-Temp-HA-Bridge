# 更新日志

## 0.2.12

- 主机登录失败时仍会通过 MQTT 显示“主机连接”为离线，不再遗留旧的在线状态。
- 新增“最近连接错误”诊断实体，显示主机登录超时、连接中断或网络地址异常等原因。

## 0.2.11

- 加固命令确认的线程同步，避免 MQTT、主机读线程与定时检查并发访问时出现竞态。
- 隔离状态回调异常，并限制异常协议帧长度，避免单个坏报文终止主机读线程或长时间阻塞解析。
- 忽略保留或非文本的 MQTT 命令，并校验总控和冬季加湿开关载荷，防止异常消息被误解释为关闭命令。

## 0.2.10

- 修复 MC7021 长时间状态流中帧边界失步后，桥接会丢弃后续状态回报并导致所有控制命令确认超时的问题。
- 异常协议日志现在只保留摘要，避免大量十六进制内容占满 Home Assistant 日志。

## 0.2.9

- 修复总控及子面板命令可能漏掉快速状态回报，随后错误显示“确认超时”的问题。
- 每次控制后主动请求一次主机状态，降低主机未主动上报时的状态不同步风险。

## 0.2.8

- 科技系统运行时，模式选择保留当前模式，不再显示为不可用。
- 新增“模式切换条件”实体，提示何时可以切换模式。

## 0.2.7

- 新增 Home Assistant 附加组件更新日志。
- CI 现在会校验每个发布版本都有对应的更新日志条目。

## 0.2.6

- 将默认 MQTT 客户端 ID 更新为 Linking The World 项目名称。
- 更新独立运行说明中的项目目录名称。

## 0.2.5

- 将附加组件目录更名为 `linking_the_world_temp_ha_bridge_addon`。
- 将附加组件 slug 更名为 `linking_the_world_temp_ha_bridge`。

## 0.2.4

- 更新 GitHub 仓库地址与附加组件仓库元数据。

## 0.2.3

- 稳定版本基线。
